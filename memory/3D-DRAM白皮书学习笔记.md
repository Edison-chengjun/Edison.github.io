# 3D DRAM堆叠与AI云端算力芯片技术白皮书
## 深度学习笔记 | 2026-03-28

---

## 一、核心结论（必背）

**白皮书核心观点：AI算力的瓶颈不在计算单元，而在"内存墙"（Memory Wall）。**

> 大模型参数从100B → 1T → 10T演进，GPU计算力增长速度 >> DRAM带宽增长速度
> → HBM 3D堆叠是当前唯一解
> → 未来必然走向GPU与DRAM的3D一体化

**技术演进阶段：**
```
PoP（手机LPDDR）
  → HBM + TSV（当前AI GPU主流）
    → 2.5D中介层（量产成熟）
      → 3DIC Hybrid Bonding（未来方向）
        → 存算一体（终极形态）
```

---

## 二、技术路线详解

### 2.1 2.5D中介层（当前主流）

**采用厂商：** NVIDIA、AMD、华为、Cerebras

| 优点 | 缺点 |
|------|------|
| 良率高 | 与GPU互连距离仍偏长 |
| 工艺成熟 | 功耗越来越难控制 |
| 可扩展性强（HBM数量可增加） | 带宽密度逼近物理极限 |
| 热设计相对容易 | |

### 2.2 3DIC（未来方向）

**目标：** HBM直接叠在GPU顶部

| 核心技术 | 说明 |
|---------|------|
| Hybrid Bonding | 直接铜-铜键合，无焊料 |
| 背面供电（BSPDN） | 新一代供电技术 |
| 低阻高密度micro-bump | 密度远超现有技术 |

**优势：**
- 互连极短
- 能耗大幅下降
- 带宽密度提升3-10倍
- 适合未来10TB/s带宽需求

**核心挑战（三大障碍）：**

| 障碍 | 说明 |
|------|------|
| **良率** | GPU良率 × HBM良率 × Bonding良率 = 成品率极低 |
| **功耗密度** | HBM发热 + GPU发热 → >1 W/mm² |
| **热阻** | GPU顶部材料限制热流通道 |

### 2.3 HBM结构深度解析

HBM组成：
- **TSV**（硅通孔）——垂直互联通道
- **Microbump**（微凸块）——层间连接
- **Base Die**（逻辑控制层）
- **DRAM层**（6/8/12/16层堆叠）

**层数增加后的挑战：**
```
8层 → 12层 → 16层
  ↓     ↓     ↓
TSV良率指数下降
机械应力增加
热流路径变长
DRAM自发热增加
```

## 三、GPU算力演进与内存架构

| 时期 | GPU单卡算力 | HBM带宽 | 架构 |
|------|------------|---------|------|
| 2010-2015 | 1-5 TFLOPS | 无HBM | 计算与内存分离 |
| 2016-2020 | 10-20 TFLOPS | 300-900 GB/s | HBM2/2E，中介层 |
| 2021-2024 | 50-80 TFLOPS | 1-2 TB/s | 2.5D成熟 |
| **2025-2030** | **100+ TFLOPS** | **4-10 TB/s** | **GPU与DRAM必须走向3D** |

## 四、海外厂商3D路线图

### 4.1 NVIDIA
```
2025-2026：引入局部Hybrid Bonding
2027-2029：GPU-SRAM/hot cache 3D堆叠
2030+：GPU-HBM直接堆叠（真正3D GPU）
```
仍坚持2.5D但路线图明显转向3D

### 4.2 AMD
Foveros样式chiplet进展领先
可能率先尝试3D DRAM堆叠

### 4.3 Intel（最激进）
- Foveros Direct技术已量产
- 最可能最早实现3D GPU + 3D DRAM

### 4.4 HBM厂商（三星/SK海力士/美光）
- 布局Hybrid Bonding HBM（HBM4之后）
- 迈向3D-DRAM + 3D-Logic混合堆叠
